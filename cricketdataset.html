ALTAIR STUDIO IMPLEMENTATION

PROJECT REPORT

Babar Azam Strike Rate Analysis Using Decision Tree

1.Objective

The primary objective of this project is to analyze the batsman (babar azam)performance data and predict whether the target has been achieved based on their strike rate. Specifically, the model utilizes a Decision Tree algorithm to classify the outcomes, establishing that if the batsman's strike rate is greater than 122 (>=122), the target is successfully achieved ('Yes'); otherwise, if the strike rate is 122 or below (<=122), the target is not achieved ('No').

2. Data Collection Source & Procedure

Source: The data is collected from a structured CSV file containing historical match statistics of the batsman.
Procedure: The CSV dataset was imported into the Altair AI Studio environment. Preprocessing and data cleansing steps were performed, and the Set Role operator was utilized to explicitly define the target label (Target_Achieved) and the predictive feature (Strike_Rate).

3. Selection and Justification of ML Technique

The machine learning technique selected for this project is Supervised Machine Learning (Classification). Justification: Since the target variable to be predicted is discrete and categorical (Yes or No), classification is the most appropriate and logically sound technique to segment the match data into distinct outcomes.

4. Selection of Specific Algorithm

The Decision Tree algorithm was selected to build the classification model. Justification: As shown in the Decision Tree automatically identified Strike_Rate as the most significant splitting feature. This algorithm is highly favored for its simplicity, visual interpretability, and its ability to easily generate human-readable IF-THEN rules that can be seamlessly translated into AI logical structures like Prolog.

5. Division of Dataset for Learning & Evaluation

"To evaluate the performance of our machine learning model accurately, the dataset was partitioned into distinct subsets for training and evaluation using the 'Split Data' operator in Altair AI Studio. As shown in the configuration, a 70/30 split ratio was implemented: 70% (0.7) of the data was allocated as the training set (Learning phase) to train the model and identify patterns, while the remaining 30% (0.3) was reserved as the testing set (Evaluation phase) to validate the model's predictive accuracy on unseen data. This standard division ensures that the model is neither underfitted nor overfitted, providing a reliable measure of its real-world performance."

6. Model Visual & Description

A. Process Workflow Architecture

Data Ingestion & Role Assignment: The dataset is loaded via the Read CSV operator, and the target label is defined using the Set Role operator to guide supervised learning.
Data Partitioning: The Split Data operator divides the dataset into training and testing subsets to ensure unbiased evaluation.
Model Training & Evaluation: The training partition is fed into the Decision Tree operator. The trained model is then duplicated via the Multiply operator, allowing it to be visualized directly while simultaneously being passed to the Apply Model and Performance operators to test its accuracy on unseen data.

B. Visual Graph Analysis

Root Node (Strike_Rate): The model identifies Strike_Rate as the absolute mathematical root of the tree. This indicates that among all available attributes in the dataset, the batsman's strike rate contains the highest Information Gain (lowest entropy), making it the most dominant predictor for classification.
Right Branch (> 122): If a batsman’s Strike Rate is strictly greater than 122, the model branches to the right, leading directly to a leaf node that predicts "Yes".
Left Branch (≤ 122): Conversely, if the Strike Rate is less than or equal to 122, the model navigates down the left branch, resulting in a leaf node prediction of "No".

C. Textual Rule Description & Leaf Node Distribution

Strike_Rate > 122: Yes {Yes=9, No=0} — This rule covers 9 total instances from the dataset, all of which fall perfectly into the positive class, confirming a clean split with zero mixing.
Strike_Rate <= 122: No {Yes=0, No=26} — This rule covers 26 instances, all of which are perfectly classified into the negative class.

8. Rule Extraction

The following explicit IF-THEN classification rules are extracted directly from the trained Decision Tree model:
1. Rule 1: IF Strike_Rate > 122 THEN Target_Achieved = 'Yes'
2. Rule 2: IF Strike_Rate <= 122 THEN Target_Achieved = 'No'

9.Rules Formation In Pure AI Logic

To bridge the gap between machine learning outcomes and computational logic, the derived decision tree rules are formalized into a language that computing environments can process natively. This logic is mapped across two distinct granularities: starting with basic conditional dependencies (Propositional Logic), and then advancing to absolute statements that universally govern every entry in the database (First-Order Logic).

Propsitional Logic

P1 : StrikeRate (High) -> TargetAchieved (Yes)
P2 : StrikeRate (Low) -> TargetAchieved (No)

First Order Logic

For all x [ StrikeRate(x, high) -> TargetAchieved(x, yes) ]
For all x [ StrikeRate(x, low) -> TargetAchieved(x, no) ]
